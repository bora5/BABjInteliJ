Reports legacy BABj ComboBox setup methods that manually create a lazy DataProvider.fromFilteringCallbacks(...) provider.

The quick fix replaces the refresh method and separate label call with createSimpleComboBox(...) or createDependentComboBox(...). For dependent ComboBoxes it also removes a listener when the listener only refreshes the child data view and clears its selection, because the BAB factory already provides that behavior. Refresh calls and their unique label calls may appear in separate groups within the same method.

The inspection also recognizes one or more standard admin-only ComboBox plus-button wrappers inside the same if (getAdmin()) block. Its quick fix replaces them with comboWithAddButton(...), makes the wrappers local, and removes the now-redundant conditionals, button fields, and imports.

The inspection intentionally ignores unsupported provider variants, refresh methods with other statements, methods with multiple call sites, and provider or add-button listeners containing custom behavior.
